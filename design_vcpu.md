# my_hypervisor 虚拟 CPU (vCPU) 模块设计与实现文档

## 1. 模块设计目标与在 Hypervisor 架构中的作用

虚拟 CPU (vCPU) 模块是 `my_hypervisor` 实现虚拟机 (Guest OS) 运行和管理的核心组件。它负责对物理 CPU 进行虚拟化，为每个 Guest OS 提供一个独立的虚拟处理器环境。在 ARMv8 (AArch64) 架构下，Hypervisor 运行在 EL2，vCPU 模块需要妥善管理 EL1/EL0 级别的 Guest OS 状态。本模块的具体设计目标和在架构中的核心作用如下：

*   **虚拟化处理器状态**：为每个 vCPU 创建并维护其完整的处理器上下文，包括通用寄存器、程序计数器 (PC)、处理器状态寄存器 (PSTATE) 和其他系统寄存器。
*   **上下文切换**：在 Hypervisor 和 Guest OS 之间，以及不同 Guest OS 之间，实现高效、安全的处理器上下文保存和恢复，确保虚拟机的平稳运行。
*   **支持 EL1/EL0 Guest OS 运行**：将物理 CPU 的 EL1/EL0 权限级别抽象给 Guest OS，并管理其在这些特权级别下的行为。
*   **提供 vCPU 管理接口**：为上层虚拟机管理模块提供创建、销毁、启动、停止 vCPU 的基本能力。
*   **与异常处理模块协同**：当 Guest OS 触发异常（如 MMIO 访问、特权指令、HVC 调用）陷入 EL2 时，vCPU 模块需要能够读取和修改 Guest OS 的寄存器状态，并决定如何处理这些异常（例如，模拟硬件行为或将控制权返回给 Guest OS）。

## 2. 文件构成与功能职责

vCPU 模块由 `vcpu.h`、`vcpu.c` 和 `vcpu_asm.S` 三个文件组成，它们共同协作完成 vCPU 的上下文管理和切换。

```text
my_hypervisor/
├── vcpu.h         # 1. vCPU 模块的公共定义、vCPU 上下文结构体和 API 声明
├── vcpu.c         # 2. vCPU 模块的 C 语言实现，负责 vCPU 的初始化和管理
├── vcpu_asm.S     # 3. vCPU 模块的汇编实现，负责底层的上下文保存和恢复
└── main.c         # 4. 集成 vCPU 模块，负责 vCPU 的创建和启动
```

### 2.1 vcpu.h (头文件)

*   **设计目的**：定义 vCPU 模块的公共接口、数据结构和常量。
*   **核心功能**：
    *   **上下文结构体定义**：定义 `vcpu_context_t` 结构体，用于存储 vCPU 的完整寄存器状态，包括通用寄存器 (`regs[31]`)、栈指针 (`sp`)、程序计数器 (`pc`) 和处理器状态 (`pstate`)。
    *   **API 声明**：`vcpu_save(vcpu_context_t *ctx)` 用于保存当前 vCPU 的上下文，`vcpu_restore(vcpu_context_t *ctx)` 用于恢复指定 vCPU 的上下文，`vcpu_init(void)` 用于初始化 vCPU 模块。

### 2.2 vcpu.c (实现文件)

*   **设计目的**：实现 `vcpu.h` 中声明的 vCPU 管理功能，包括 vCPU 的初始化和状态维护。
*   **核心功能**：
    *   **vCPU 结构体初始化**：在 `vcpu_init` 函数中，初始化全局 `vcpu_context_t` 结构体，设置默认的栈指针、程序计数器 (指向 Guest OS 的入口点) 和处理器状态 (例如，EL1h 模式)。
    *   **通用寄存器清零**：在初始化时将 vCPU 的所有通用寄存器清零，确保一个干净的启动环境。

### 2.3 vcpu_asm.S (汇编实现文件)

*   **设计目的**：提供底层的汇编代码，负责在 Hypervisor 模式 (EL2) 下高效地保存和恢复 Guest OS (EL1/EL0) 的处理器上下文。
*   **核心功能**：
    *   **`vcpu_save` 函数**：将当前物理 CPU 的通用寄存器 (`x0-x30`)、栈指针 (`sp`)、程序计数器 (`pc`，即 `elr_el2`) 和处理器状态 (`pstate`，即 `spsr_el2`) 保存到 `vcpu_context_t` 结构体中。
    *   **`vcpu_restore` 函数**：从 `vcpu_context_t` 结构体中恢复通用寄存器、栈指针、程序计数器和处理器状态到物理 CPU，并最终通过 `eret` 指令将控制权切换到 Guest OS。

## 3. 核心代码逻辑与目的（作为架构师的审视）

### 3.1 完整的上下文保存与恢复

*   **审视与目的**：**vCPU 上下文的完整性和精确性是实现可靠虚拟化的基石**。`vcpu_asm.S` 中的 `vcpu_save` 和 `vcpu_restore` 函数必须保存和恢复所有的 Guest OS 可访问寄存器，包括通用寄存器和关键的系统寄存器（通过 ELR_EL2 和 SPSR_EL2 间接管理 PC 和 PSTATE）。任何遗漏都可能导致 Guest OS 状态不一致或崩溃。这种汇编层面的精确控制是 C 语言难以直接实现的。
*   **实现细节**：
    *   `vcpu_save`：利用 `stp` 指令高效地将成对的通用寄存器保存到内存。通过 `mrs` 指令读取 `ELR_EL2` 和 `SPSR_EL2`。
    *   `vcpu_restore`：利用 `ldp` 指令恢复通用寄存器。通过 `msr` 指令写入 `ELR_EL2` 和 `SPSR_EL2`，并通过 `eret` 指令实现异常返回，从而将控制权转移到 Guest OS。

### 3.2 vCPU 初始状态设置

*   **审视与目的**：`vcpu_init` 函数负责设置 vCPU 的初始状态。**一个合理的初始状态对于 Guest OS 的成功启动至关重要**。例如，将 `vcpu_ctx.pc` 设置为 Guest OS 的入口地址，`vcpu_ctx.pstate` 设置为 EL1h 模式（EL1 with SP_ELx），可以确保 Guest OS 在正确的特权级别和入口点启动。

### 3.3 与异常处理模块的协同

*   **审视与目的**：vCPU 模块与中断与异常管理模块 (`irq.c`, `start.S`) 紧密耦合。当 Guest OS 触发异常陷入 EL2 时，`irq.c` 中的 `handle_exception` 会接收到 `struct cpu_regs *regs`，其中包含了 Guest OS 的上下文。**vCPU 模块需要能够利用这个上下文来模拟 Guest OS 的行为，或者在处理完异常后，将修改后的上下文恢复给 Guest OS**。例如，HVC 处理程序会修改 `regs->elr_el2` 来跳过 HVC 指令，然后通过 `eret` 返回到 Guest OS。

## 4. 虚拟 CPU (vCPU) 模块设计大纲（Design Outline）

```text
+--------------------------------------------------------------------------+
|                      Hypervisor main() 函数启动                         |
+--------------------------------------------------------------------------+
                                     |
                                     v
+--------------------------------------------------------------------------+
|                         vcpu_init() 调用                                 |
|               (初始化全局 vcpu_context_t 结构体)                        |
+--------------------------------------------------------------------------+
                                     |
                                     v
+--------------------------------------------------------------------------+
|                   设置 vcpu_ctx.sp (Guest OS 初始栈指针)                 |
+--------------------------------------------------------------------------+
                                     |
                                     v
+--------------------------------------------------------------------------+
|                   设置 vcpu_ctx.pc (Guest OS 入口点)                     |
+--------------------------------------------------------------------------+
                                     |
                                     v
+--------------------------------------------------------------------------+
|                   设置 vcpu_ctx.pstate (Guest OS 初始处理器状态，EL1h)     |
+--------------------------------------------------------------------------+
                                     |
                                     v
+--------------------------------------------------------------------------+
|                     清零 vcpu_ctx.regs (通用寄存器)                       |
+--------------------------------------------------------------------------+
                                     |
                                     v
+--------------------------------------------------------------------------+
|                          vcpu_init() 返回                                 |
+--------------------------------------------------------------------------+
                                     |
                                     v
+--------------------------------------------------------------------------+
|                     Hypervisor 继续初始化，最终启动 Guest OS            |
+--------------------------------------------------------------------------+
```

### 4.1 虚拟 CPU (vCPU) 模块详细步骤

1.  **vCPU 模块初始化** (`vcpu_init`):
    *   在 `main` 函数中调用 `vcpu_init()`。
    *   初始化 `vcpu_context_t vcpu_ctx` 全局结构体。
    *   将 `vcpu_ctx.sp` 设置为 Guest OS 的初始栈指针地址。
    *   将 `vcpu_ctx.pc` 设置为 Guest OS 的入口点地址。
    *   将 `vcpu_ctx.pstate` 设置为 Guest OS 初始的处理器状态 (例如，EL1h 模式)。
    *   将 `vcpu_ctx.regs` 数组清零。

2.  **vCPU 上下文保存** (`vcpu_save`):
    *   当 Guest OS 触发异常陷入 EL2 时，由异常处理模块调用 `vcpu_save(vcpu_context_t *ctx)`。
    *   在 `vcpu_asm.S` 中，使用 `stp` 指令保存通用寄存器 `x0-x30` 到 `ctx->regs`。
    *   使用 `mrs` 指令保存 `SP_EL0` 到 `ctx->sp_el0` (尽管 `vcpu_context_t` 中没有 `sp_el0` 字段，但在实际完整的实现中会保存)。
    *   使用 `mrs` 指令保存 `ELR_EL2` 到 `ctx->elr_el2`，这代表了 Guest OS 的 `pc`。
    *   使用 `mrs` 指令保存 `SPSR_EL2` 到 `ctx->spsr_el2`，这代表了 Guest OS 的 `pstate`。

3.  **vCPU 上下文恢复** (`vcpu_restore`):
    *   当 Hypervisor 准备将控制权交还给 Guest OS 时，调用 `vcpu_restore(vcpu_context_t *ctx)`。
    *   在 `vcpu_asm.S` 中，使用 `ldr` 或 `ldp` 指令从 `ctx` 恢复通用寄存器 `x0-x30`。
    *   使用 `msr` 指令恢复 `SP_EL0`、`ELR_EL2` 和 `SPSR_EL2`。
    *   执行 `eret` 指令，将控制权从 EL2 返回到 Guest OS (EL1/EL0)，并恢复其处理器状态。
