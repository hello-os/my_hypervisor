# my_hypervisor 模块缺失与规划

根据对 Xvisor 架构的对比，`my_hypervisor` 目前仅具备最基础的启动、内存映射、初步中断处理、vCPU 基础上下文切换及调度器框架。为了达到类似 Xvisor 的功能，建议按以下顺序补充核心模块：

## 1. 核心模块规划 (优先级顺序)
- [ ] **IOMMU 驱动**: 为设备隔离和安全性提供硬件支持。
- [ ] **中断管理 (GIC 完整驱动)**: 目前仅有初步处理，需扩展至支持所有 GIC 特性、中断路由及虚拟中断注入。
- [ ] **平台驱动框架**: 实现统一的设备驱动注册与管理接口。
- [ ] **设备模拟 (Emulation)**: 实现 UART、Timer 等常用外设的虚拟化模拟。
- [ ] **客户机 (Guest) 管理**: 实现 Guest 镜像的加载、虚拟地址空间配置及 VM 运行控制。
- [ ] **动态内存管理**: 目前为静态分配，需实现动态堆管理 (Malloc/Free)。
- [ ] **设备树 (DT) 解析**: 从 QEMU 传递的 DTB 中动态发现系统资源。

## 2. 下一步执行计划
1.  **实现动态内存管理 (vmm_heap)**: 这是后续所有模块的基础。
2.  **完善设备树 (DT) 解析**: 实现对系统内存、外设地址的自动化发现。
3.  **扩展中断管理**: 实现中断共享、虚拟中断注入等。

---
请确认上述顺序，我们将优先实现 **动态内存管理 (vmm_heap)**。
