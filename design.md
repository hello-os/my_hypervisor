# my_hypervisor 架构设计与启动流程实现文档

本项目是一个面向 **ARMv8 (AArch64)** 架构的极简、高可靠性的 Hypervisor（虚拟化管理程序）原型系统。其设计参考了 **Xvisor** 的核心机制，如：
- **裸金属（Bare-metal）架构**：不依赖任何第三方操作系统（如 Linux/RTOS）或标准库，直接在硬件层（EL2 / EL3）运行。
- **EL2 级特权运行**：利用 ARMv8 虚拟化扩展提供的 Hypervisor 模式（Exception Level 2）管理底层的物理资源与上层的虚拟机（VM）。

本文档将详细阐述项目的文件构成、代码编写逻辑、设计目的以及整个启动功能的设计大纲。

---

## 1. 为什么创建这些文件？（文件结构及作用）

在裸金属开发中，我们需要对编译、链接、引导和执行的每个细节进行精确控制。因此，本项目构建了以下四个核心文件：

```text
my_hypervisor/
├── Makefile       # 1. 跨平台编译构建文件：定义如何将源码转换成目标二进制文件
├── linker.ld      # 2. 链接脚本：规划代码和数据在物理内存中的精确布局
├── start.S        # 3. 引导汇编代码：处理芯片上电或 bootloader 跳转后的硬件初始化
└── main.c         # 4. 核心 C 入口：初始化系统并运行 Hypervisor 核心调度与逻辑
```

### 1.1 Makefile
*   **设计目的**：实现自动化跨平台编译。由于宿主机（通常是 x86_64）与目标机（ARM64）架构不同，必须使用交叉编译器（`aarch64-linux-gnu-`）。
*   **核心功能**：
    *   通过 `-ffreestanding` 告诉编译器环境不存在标准库。
    *   通过 `-nostdlib` 和 `-nostartfiles` 屏蔽标准链接库和默认的引导代码（我们自己提供）。
    *   将生成的 `elf` 格式二进制文件转换成适合裸金属直接加载的 flat binary 格式 (`.bin`)。

### 1.2 linker.ld
*   **设计目的**：裸金属程序没有操作系统为其重定位地址，因此加载地址必须固定。
*   **核心功能**：
    *   将入口点设置为 `_start`。
    *   设置起始物理地址为 `0x40080000`（QEMU `virt` 架构的标准内核加载地址）。
    *   确保关键的引导段 `.text.boot` 始终排在最前面，确保上电时 `_start` 汇编指令首先被执行。
    *   规划 BSS 段的边界，以便在运行时将其清零；分配 16KB 的独立引导栈，确保 C 函数在执行时能正常压栈。

### 1.3 start.S
*   **设计目的**：由于 C 语言运行时需要特定的硬件状态（如栈指针、BSS 零初始化等），必须使用汇编语言进行第一阶段硬件级初始化。
*   **核心功能**：
    *   屏蔽全局中断（DAIF 寄存器）。
    *   支持安全状态切换。如果系统在 EL3（Secure/Monitor 模式）启动，将其安全平滑地降级到非安全 EL2（Hypervisor 模式），并对相应的关键系统寄存器进行配置。
    *   关闭 MMU 和一级缓存，以便在没有映射表的情况下，C 代码可以安全读写物理内存。
    *   初始化 SP 寄存器并将 BSS 清零。

### 1.4 main.c
*   **设计目的**：使用高级 C 语言接管系统的后续管理。
*   **核心功能**：
    *   实现极简的 PL011 串口驱动，绕过复杂的内核接口直接操作 MMIO 寄存器（`0x09000000`）。
    *   读取系统状态寄存器并向用户控制台输出 Hypervisor 的关键诊断信息，包含 Exception Level 状态及引导寄存器值。
    *   提供最终的循环死循环作为 Hypervisor 核心调度回路的入口。

## 2. 为什么这样写代码？（核心代码逻辑与目的）

### 2.1 Exception Level 的安全跃迁与检测 (start.S)
```assembly
    mrs x4, currentel
    and x4, x4, #0xc
    cmp x4, #0xc    /* 检测当前是否处于 EL3 */
    b.eq el3_entry
    cmp x4, #0x8    /* 检测当前是否处于 EL2 */
    b.eq el2_entry
```
*   **为什么这样做**：ARMv8 有四个异常级别（EL0 - EL3）。
    *   **EL3**：安全监视器模式（Secure Monitor）。
    *   **EL2**：Hypervisor 模式。
    *   如果 QEMU 在 EL3 启动，我们必须配置 `SCR_EL3`（设置 NS=1 开启非安全态，设置 RW=1 开启 64位下级异常，设置 HCE=1 启用 EL1 对 EL2 的 hypercall 拦截呼叫）。
    *   通过填充 `SPSR_EL3` 设定目标状态为 `EL2h`（即 `0x3c9`），然后利用 `eret` 指令在硬件层执行安全跳转，完成降级。

### 2.2 SCTLR_EL2 系统控制寄存器的初始化 (start.S)
```assembly
    mrs x0, sctlr_el2
    bic x0, x0, #(1 << 0)   /* 关闭 MMU */
    bic x0, x0, #(1 << 2)   /* 关闭 D-Cache */
    bic x0, x0, #(1 << 12)  /* 关闭 I-Cache */
    bic x0, x0, #(1 << 1)   /* 关闭 内存对齐检查 */
    msr sctlr_el2, x0
    isb
```
*   **为什么这样做**：在上电瞬间，MMU 和缓存的状态是未知的。如果 MMU 处于开启状态，所有的内存访问都将被当作虚拟地址进行转换，导致页表缺失异常。因此，最安全的办法是在引导之初彻底关闭 MMU、D-Cache 与对齐检查，确保整个系统运行在确定性的裸内存读写模式下。

### 2.3 BSS 段清零与栈设置 (start.S & linker.ld)
```assembly
    ldr x0, =__bss_start
    ldr x1, =__bss_end
bss_loop:
    cmp x0, x1
    b.ge bss_done
    str xzr, [x0], #8
    b bss_loop
```
*   **为什么这样做**：
    *   **BSS 清零**：C 语言规范要求未初始化的全局变量和静态变量初始值为 0。如果没有汇编引导代码去清零 BSS，这些变量在物理内存中会呈现随机脏数据，导致 C 代码运行逻辑崩溃。
    *   **栈初始化**：C 语言的局部变量分配、函数调用参数传递、寄存器上下文保存都依赖栈。所以必须在 `bl main` 之前，将堆栈指针 `sp` 指向链接脚本中预留的 `__stack_top` 区域。

### 2.4 PL011 硬件串口 MMIO 驱动 (main.c)
```c
#define PL011_UART_BASE 0x09000000

static void uart_putc(char c)
{
    volatile unsigned int *uart = (volatile unsigned int *)PL011_UART_BASE;
    *uart = c;
}
```
*   **为什么这样做**：QEMU `virt` 机器的物理 UART 设备映射在 `0x09000000`。在没有复杂驱动框架时，最简单有效的方法就是直接进行裸地址写（volatile 保证编译器不会优化掉该写操作），这为裸金属调试提供了一条极为可靠的“日志生命线”。

## 3. 启动功能设计大纲（Design Outline）

整个启动流从机器上电（物理或虚拟 CPU 启动）开始，至进入 C 语言循环为止。整体流向图如下所示：

```text
                  +--------------------------------+
                  |         QEMU / HW 上电          |
                  +--------------------------------+
                                  |
                                  v
                  +--------------------------------+
                  |      _start (屏蔽全局中断)      |
                  +--------------------------------+
                                  |
                                  v
                  +--------------------------------+
                  |      读取 CurrentEL 寄存器      |
                  +--------------------------------+
                                  |
               +------------------+------------------+
               |                                     |
               v (EL3)                               v (EL2)
    +----------------------+               +----------------------+
    |    配置 SCR_EL3      |               |  SCTLR_EL2 安全初始化 |
    |   设置 SPSR_EL3      |               |  (禁用 MMU, Caches)  |
    |      eret 跳转       |               +----------------------+
    +----------------------+                           |
               |                                       |
               +------------------+--------------------+
                                  |
                                  v
                  +--------------------------------+
                  |    设置堆栈 SP = __stack_top    |
                  +--------------------------------+
                                  |
                                  v
                  +--------------------------------+
                  |    清除 BSS 段 (__bss_start)    |
                  +--------------------------------+
                                  |
                                  v
                  +--------------------------------+
                  |    调用 main(r0, r1, r2, r3)   |
                  +--------------------------------+
                                  |
                                  v
                  +--------------------------------+
                  |  UART 输出欢迎信息与系统诊断状态  |
                  +--------------------------------+
                                  |
                                  v
                  +--------------------------------+
                  |     进入 while(1) 核心主循环    |
                  +--------------------------------+
```

### 3.1 引导大纲步骤明细

1.  **第一阶段：底层屏蔽与态检测（汇编段）**
    *   **步骤 1**：CPU 复位或引导加载程序加载二进制，硬件 PC 跳转至 `0x40080000`（执行 `_start`）。
    *   **步骤 2**：通过 `msr daifset, #0xf` 瞬间关闭全局异步异常（FIQ、IRQ、SError 和 Debug），确保在完全初始化前不被外界中断打断。
    *   **步骤 3**：通过 `currentel` 寄存器判断当前的 Exception Level。如果发现系统处于 EL3，则通过配置 `scr_el3` 打开 EL2 特权通道，利用 `eret` 硬件机制下跃到非安全 EL2。

2.  **第二阶段：环境准备（汇编段）**
    *   **步骤 4**：重置 `sctlr_el2` 控制寄存器，强制关闭 EL2 阶段的 MMU、数据和指令 Cache，从而在初始化页表前处于直接物理寻址状态。
    *   **步骤 5**：通过链接脚本提供的 `__stack_top`，向 SP 寄存器写入栈地址，搭建 C 运行环境的物理栈帧。
    *   **步骤 6**：计算 `__bss_start` 到 `__bss_end` 的边界，通过双寄存器自增机制，对整个 BSS 段写入 64 位的 0 值。

3.  **第三阶段：高级初始化与主循环（C 语言段）**
    *   **步骤 7**：使用 `bl main` 跳转到 C 语言主入口，寄存器 `x0` - `x3`（如 QEMU 传递的设备树 FDT 物理地址等）将透明地传递为 `main` 函数的四个参数 `r0` - `r3`。
    *   **步骤 8**：在 C 中初始化 PL011 串口指针，向物理 UART 发送字节序列输出系统诊断字符。
    *   **步骤 9**：动态读取 `CurrentEL` 系统寄存器并输出，向开发者反馈 Exception Level 的精确物理状态。
    *   **步骤 10**：进入内核的主循环 `while (1)`，保持 CPU 在极低开销状态，等待中断管理及下阶段虚拟机监控、调度模块的注入。
