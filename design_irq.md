# my_hypervisor 中断与异常管理模块设计与实现文档

## 1. 模块设计目标与在 Hypervisor 架构中的作用

中断与异常管理模块是 `my_hypervisor` 的另一大核心基石。在 AArch64（ARMv8 64位）架构下，Hypervisor 运行在 Exception Level 2 (EL2)，负责统筹控制上层虚拟机（EL1/EL0）的软硬件异常与底层硬件中断。本模块的具体设计目标和在架构中的核心作用如下：

*   **接管 EL2 级异常向量表**：构建符合 ARMv8 规范的 2048 字节对齐的向量表，全面捕获 EL2 自身运行或下级异常模式（Guest OS）触发的同步异常、IRQ、FIQ 和 SError。
*   **物理中断控制器 (GICv2) 统筹**：接管并配置 ARM 通用中断控制器（Generic Interrupt Controller, GICv2），对 Distributor（分发器）和 CPU Interface（CPU 接口）进行高精度配置，完成物理中断的使能、分发及核间投递。
*   **支持虚拟化陷入与模拟机制 (Trap-and-Emulate)**：提供 Hypervisor Call (HVC) 异常处理器。当 Guest OS 试图发起虚拟机管理呼叫（如向 Hypervisor 请求服务或特权操作）时，通过 HVC 陷入 EL2，实现虚拟化的核心逻辑。
*   **精确的定时中断服务**：初始化 ARMv8 物理定时器（Generic Timer），利用周期性的定时中断为 Hypervisor 提供高精度的调度时钟源与时间片控制机制。
*   **高可靠的现场现场保护与恢复**：在异常发生的瞬间，完整保存物理 CPU 的全套寄存器上下文（x0-x30 及状态寄存器），并能在处理完毕后安全无缝地恢复，保障 Guest OS 或 Hypervisor 自身的连续运行。

## 2. 文件构成与功能职责

中断与异常管理模块由两个核心 C 文件（`irq.h` 和 `irq.c`）和汇编入口代码（`start.S`）共同构成，它们与系统初始化模块（`main.c`）进行高效交互。

```text
my_hypervisor/
├── irq.h          # 1. 中断控制和系统寄存器的宏定义、现场帧结构体及 API 声明
├── irq.c          # 2. GIC 初始化、定时器配置及 C 语言第一级异常分发器
├── start.S        # 3. 提供异常向量表结构、save_regs / restore_regs 上下文保护宏
└── main.c         # 4. 调用初始化函数，并通过控制状态寄存器使能全局中断
```

### 2.1 irq.h (头文件)

*   **设计目的**：定义 GICv2 硬件寄存器偏移量、异常分类（ESR_EL2.EC）、寄存器现场结构体模型并公开公共 API。
*   **核心功能**：
    *   **地址定义**：定义 QEMU 虚拟平台标准的 `GICD_BASE` (0x08000000) 与 `GICC_BASE` (0x08010000) 控制器映射基址。
    *   **寄存器偏移**：包括分发器控制、使能、挂起、优先级、目标核寄存器偏移，以及 CPU 接口控制、掩码、应答、结束中断寄存器偏移。
    *   **EC 异常类别码**：定义 `EC_DATA_ABORT_EL1`、`EC_HVC` 等，用于精确解析 `ESR_EL2`。
    *   **上下文结构体定义**：定义 `struct cpu_regs`，映射保存在栈上的全部 31 个通用寄存器以及系统管理寄存器（SPSR_EL2, ELR_EL2, ESR_EL2 等）。

### 2.2 irq.c (实现文件)

*   **设计目的**：实现 GIC 控制器初始化、定时器重载、以及 C 阶段的中断异常派遣。
*   **核心功能**：
    *   **GIC Distributor 初始化 (`gic_dist_init`)**：禁用分发器后，根据 `GICD_TYPER` 动态计算支持的中断线条数。对所有中断进行去能与清除挂起状态，将所有中断优先级设为 `0xF0` 并重定向至 CPU0 目标核心，最后全局使能 Group 0 / Group 1 中断接收。
    *   **GIC CPU Interface 初始化 (`gic_cpu_init`)**：配置 `GICC_PMR` 优先级掩码为最低门槛 `0xFF`，保证所有优先级的中断均能打通。最后使能 CPU0 的中断接收控制位。
    *   **通用定时器驱动 (`timer_init`)**：读取 `cntfrq_el0`（硬件时钟频率），根据指定的微秒级间隔计算滴答数，将滴答数写入 `cntv_tval_el0` 寄存器并使能 EL2 物理定时器。
    *   **C 第一级中断分发器 (`gic_irq_handler`)**：读取 `GICC_IAR` 寄存器获取发生的中断 ID。若收到 ID 为 27（物理定时器中断），输出对应调试信息，重装载定时计数器。最后向 `GICC_EOIR` 寄存器写入中断 ID 以清空该中断的状态。
    *   **陷入派遣器 (`handle_exception`)**：解析 `ESR_EL2` 的 Exception Class。若为 `EC_HVC`，则路由至 `hvc_handler`，执行下级虚拟机的 Hypercall 处理，并将 ELR_EL2 自动自增 4 字节以跳过陷入指令；否则，路由至 `default_exception_handler` 进行现场打印并挂起系统。

### 2.3 start.S (向量表与汇编上下文)

*   **异常向量表 (`el2_vector_table`)**：提供 2048 字节对齐的大表。由于 AArch64 架构将异常分为 4 个来源与 4 种异常类型，因此规划了 16 个对齐的入口。
*   **上下文宏 (`save_regs` / `restore_regs`)**：在每一个中断入口被物理触发时，首先利用 SP 减去足够空间，执行 `stp` 指令保存通用寄存器，同时读取 `sp_el0`、`elr_el2`、`spsr_el2` 等敏感寄存器转存入 `struct cpu_regs` 对应位置；执行完 C 语言派遣后，再原子地利用 `restore_regs` 恢复 CPU 原始状态并执行 `eret` 退出异常。

### 2.4 main.c (初始化与中断启用)

*   在 `main` 函数启动流程的后期，顺序调用 `irq_init()` 完成 GIC 配置和时钟设定。
*   最后，调用汇编实现的 `irq_enable()`，修改 PSTATE.A 状态位清空屏蔽字，允许硬件中断正式传入。

## 3. 核心代码逻辑与目的（作为架构师的审视）

### 3.1 异常处理的二级架构设计 (start.S 汇编现场 & irq.c 高级分发)

*   **审视与目的**：在裸金属虚拟化开发中，**最容易出错的环节是异常入口处的寄存器状态污染**。如果直接从汇编跳进 C 函数，编译器生成的序言代码会立即破坏 `x0-x7` 等传参寄存器和 `lr` (x30)。因此，构建 `save_regs` 宏在最外层实行全寄存器压栈是最安全和稳健的做法。
*   将栈指针传递给 C 函数 `void handle_exception(struct cpu_regs *regs)`，使 C 代码能够通过指针直接读写被陷入虚拟机的上下文，极大地方便了 HVC 返回值的注入（直接修改 `regs->x0`）和 HVC 指令的跳过（将 `regs->elr_el2 += 4`）。

### 3.2 GICv2 级联管理机制与多核前瞻 (`gic_dist_init`, `gic_cpu_init`)

*   **审视与目的**：GICv2 采用分级控制理念：
    *   **Distributor (GICD)** 属于全局性质，初始化一次即可。我们通过禁用它来防止在重置中断线属性时产生杂波或假中断，而后计算 `num_irq_lines` 并彻底禁能和清除中断使能状态，将全部中断的默认目标配置为 CPU0（`0x01010101`），这是单核原型最稳定的基线。
    *   **CPU Interface (GICC)** 属于核私有级。在进入多核架构时，每个 CPU 核心必须各自执行一次 `gic_cpu_init`。我们在这个子模块中拉低优先级掩码，保证不漏掉任何外部物理事件。

### 3.3 HVC (Hypervisor Call) 陷入拦截与控制台反馈

*   **审视与目的**：当 Guest 操作系统发出 `hvc` 指令后，硬件将 PC 放入 `ELR_EL2`，CPU 状态放入 `SPSR_EL2`，并将服务 ID 装在 `x0` 里。本模块的设计能立即在串口打印 `[HVC] Hypervisor Call from EL1!` 并回显 HVC 编码。这种直观的调试机制对构建客户操作系统（Guest OS）起到了强有力的支柱作用。

+--------------------------------------------------------------------------+
                                     |
                                     v
+--------------------------------------------------------------------------+
|                  汇编 save_regs 宏 (保存 CPU 上下文)                       |
+--------------------------------------------------------------------------+

                                     |

+--------------------------------------------------------------------------+

### 4.1 中断与异常管理模块详细步骤

1.  **GIC 与时钟总初始化** (`irq_init`):
    *   在 Hypervisor 核心 C 初始化代码（`main.c`）中顺序激活。
    *   首先运行分发器自检与重置（`gic_dist_init`）。
    *   随后开启 CPU 局部的中断接口（`gic_cpu_init`）。
    *   最后拉起高精度计时器（`timer_init`），输入 10ms 周期参数，打印初始化成功日志。

2.  **配置全局分发器** (`gic_dist_init`):
    *   写入 `GICD_CTLR = 0` 暂时锁定总线。
    *   读出 `GICD_TYPER`，提取其 `[4:0]` 位，自加 1 并乘以 32 得到中断引脚总数。
    *   通过写 `GICD_ICENABLER` 和 `GICD_ICPENDR` 消除残留的任何中断使能与挂起。
    *   通过 `GICD_IPRIORITYR` 配置每个中断为 `0xF0`。
    *   通过 `GICD_ITARGETSR` 绑定所有外设到 CPU0。
    *   向 `GICD_CTLR` 写入 1 彻底启动分发。

3.  **配置本地 CPU 接口** (`gic_cpu_init`):
    *   向 `GICC_PMR` 写入 `0xFF` 开启全范围的中断优先级准入。
    *   向 `GICC_CTLR` 写入 1 正式放行 CPU 核心接收硬件 IRQ 信号。

4.  **初始化周期时钟** (`timer_init`):
    *   通过汇编指令 `mrs` 提取 `cntfrq_el0` 中的系统主频 ticks。
    *   将 ticks 与目标延迟微秒数（如 10000us）换算，得到重置装载值。
    *   将此值写入虚拟时间间隔寄存器 `cntv_tval_el0`。
    *   将 1 写入 `cntv_ctl_el0` 启动定时器，并在到期时抛出 IRQ 27。

5.  **全局解除屏蔽** (`irq_enable`):
    *   通过修改当前处理机状态：`msr daifclr, #2` 恢复 CPU 响应外部 IRQ 中断的硬件能力。
